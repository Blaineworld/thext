Thext's Code Standards

Thext's internal code standards are…

Use of proper emoticons!
If a smiley face is present in the source code, it shouldn't be written as :), rather, it should be written as :]; it's more smiley that way.
Keep it under 100 columns!
Thext's source code and documentation generally try to stay under 100 columns, because we, being non-plumbers, want to stay on the left side of the screen. That being said, it might sometimes go over and that's okay.
Hold your curly braces close!
Thext's source code puts curly braces on the same lines as their statements. This is both because it looks nice and to avoid confusion with standalone code blocks.
Double quotes, please!
Thext mainly uses double quotes, because we are filthy Americans. We do use double quotes for special cases, though, like 'use strict'.
